# Plutus · 基于 Hermes Agent 的自动记账软件

Plutus 自动采集招商银行交易邮件，使用 Hermes Agent 解析和分类，生成本地账本，并通过微信完成通知、查询和分类纠正。

## 为什么需要 Plutus？

银行卡、信用卡、微信、支付宝每天都在产生流水——但你想真正知道“这个月到底花了多少钱”，普通记账工具很快就不够用了：

- 🧾 “信用卡消费了一笔淘宝订单，里面其实买了 3 个商品” → **只能记成一笔**，后面退一个商品就乱了
- 🔁 “这个月买了东西，下个月退款了” → **月度统计被冲乱**，消费月虚高，退款月变负
- 💳 “信用卡先付押金，商家后来退了一部分” → **算不清真实花费**，押金和消费混在一起
- 🍽️ “团队聚餐我先垫付，后来公司报销到借记卡” → **消费和进项对不上**，不知道这笔该不该算自己的支出
- 🏷️ “同一个商户，有时是餐饮，有时是孩子，有时是工作” → **固定规则不够用**，分类越记越累
- 📆 “信用卡账单不是自然月，是每月 15 号到次月 14 号” → **按自然月统计没意义**，对不上真实账单
- 📥 “银行邮件、交易提醒、报销到账都散在不同地方” → **要手动录入**，漏一笔统计就不准
- 💬 “每次分类都要打开 App 改” → **操作成本高**，最后又懒得记
- 🔐 “财务数据放到第三方平台不放心” → **隐私边界不清楚**，账本数据不在自己手里

**这些不是简单的“记一笔账”问题，而是真实生活里的账务整理问题。**

普通记账软件擅长记录流水，但不擅长理解流水背后的关系：买了又退、部分退款、押金退回、垫付报销、跨月账单、商户分类偏好……这些都需要你自己手工处理。

**Plutus 把这件事变成自动化账本：**

```text
银行邮件自动采集 → Hermes AI 解析和分类 → 本地账本统计 → 微信通知和纠正
```

它不是让你每天手动记账，而是帮你把银行真实流水整理成“你真正关心的消费”。

**安装也是一句话：**

```text
帮我安装 Plutus：https://raw.githubusercontent.com/zxh2010/plutus/main/docs/install.md
```

安装完成后，Plutus 会自动采集招商银行信用卡、借记卡和进项邮件，生成本地账本，并通过 Hermes/微信让你确认、纠正和查询。

### ✅ Plutus 解决了什么

| | |
|---|---|
| 🧠 **AI 分类** | Hermes 根据分类体系和你教过的知识判断分类。你纠正得越多，它越懂你的记账习惯 |
| 🔁 **退款抵消** | 买了又全额退款的交易会自动抵消，不污染本月消费统计 |
| ✂️ **交易拆分** | 一笔淘宝/京东订单可以拆成多笔真实消费，后续部分退款只抵消对应部分 |
| 🔗 **押金合并** | 支出和后续部分退款可以合并成净消费，跨月退款也能归到正确月份 |
| 🍽️ **报销核对** | 自动识别工资、报销、汇款等进项，方便核对垫付款是否到账 |
| 📆 **账单周期** | 支持按信用卡账单周期统计，不只按自然月统计 |
| 💬 **微信交互** | 新交易推送到微信，分类不准可以直接纠正，Plutus 会记住 |
| 🔐 **本地账本** | 消费流水和邮箱授权码只保存在本机，没有服务器保存用户数据 |

> Plutus 不是帮你“记录流水”，而是帮你算清楚真实消费。

## 1. 原理

```text
招商银行交易邮件
        │
        ▼
邮箱 IMAP 增量采集
        │
        ├── 信用卡「每日信用管家」→ 交易解析
        └── 借记卡「一卡通账户变动通知」→ Hermes AI 识别支出、转账和进项
        │
        ▼
SQLite 本地账本
        │
        ├── Web 控制台：明细、分类、统计、进项、知识库
        └── Hermes → 微信：新交易通知、查询和分类纠正
```

核心数据只保存在本机：

- 邮箱通过 IMAP 只读采集，支持 QQ 邮箱、163 邮箱和 Gmail，并使用消息 ID 和 UID 水位增量去重。
- Plutus 不上传消费明细，也没有服务器保存用户的交易流水。
- 采集邮件所需的邮箱授权码只保存在本地 `~/.plutus/secrets/mail_auth.json`。
- Plutus 在本地采集和解析邮件；Hermes 负责 AI 分类、微信通知、查询和纠正。
- SQLite 是账本的唯一事实来源。
- Web 控制台和 Hermes MCP 通过同一套本地 API 读写账本。
- 用户在微信中的查询或纠正，由 Hermes 调用 Plutus MCP 写回本地数据库。

## 2. 一键安装 / 更新

前置条件：

- macOS
- Python 3.10+
- Git
- 已安装 Hermes Agent，并已配置微信通道
- 已有 QQ 邮箱、163 邮箱或 Gmail，用于接收招商银行交易邮件
- 招商银行已开启“一卡通账户变动通知”和“每日信用管家”的 Email 通知

首次安装，把下面这句话发给 Hermes：

```text
帮我安装 Plutus：https://raw.githubusercontent.com/zxh2010/plutus/main/docs/install.md
```

后续更新，把下面这句话发给 Hermes：

```text
帮我更新 Plutus：https://raw.githubusercontent.com/zxh2010/plutus/main/docs/install.md
```

Hermes 会自动安装或更新：

1. 将 Plutus 安装到 `~/.plutus`。
2. 创建 Python 虚拟环境和本地配置。
3. 初始化 SQLite 账本。
4. 注册 Plutus MCP。
5. 安装并启动 daemon 与 Web 服务。
6. 检查 Hermes 微信目标、数据库和服务状态。

更新时会执行 `git pull --ff-only` 和安装自检，但不会覆盖本地数据：

- `~/.plutus/config.toml`
- `~/.plutus/plutus.db`
- `~/.plutus/secrets/mail_auth.json`

安装完成后，打开：

```text
http://127.0.0.1:8973/#config
```

在本地设置页选择邮箱类型，填写邮箱地址和授权码，再运行邮件自检即可。设置页会按 QQ 邮箱、163 邮箱或 Gmail 显示对应授权入口；授权码只保存在 `~/.plutus/secrets/mail_auth.json`，无需发送给 Hermes。

完整安装规则和安全边界见 [`docs/install.md`](docs/install.md)。
